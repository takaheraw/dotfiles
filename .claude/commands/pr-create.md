## PR Create

変更分析に基づく Draft PR の自動作成。

### 手順

1. **ブランチの確認・作成**
   - 現在のブランチを確認し、デフォルトブランチ（main/master/develop 等）上にいる場合はフィーチャーブランチを作成
   - デフォルトブランチの最新を pull してからブランチを切る
   - 既にフィーチャーブランチ上にいればスキップ

2. **コミットの分割**
   - 未コミットの変更がある場合は `/semantic-commit` の手順でセマンティックなコミットに分割
   - 既に複数の適切なコミットがあればスキップ

3. **プッシュ**
   - 未プッシュの場合は `-u` フラグ付きでプッシュ

4. **変更内容の分析**
   - `git diff <base>...HEAD` で全変更を確認
   - 変更ファイル・内容からタイトルと説明文を生成

5. **説明文の作成**
   - `.github/PULL_REQUEST_TEMPLATE.md` があれば使用（なければデフォルト構成）
   - 既存内容は一切変更しない（空セクションのみ補完）
   - HTML コメント・区切り線は完全保持

6. **ラベルの自動選択（最大 3 個）**
   - 既存ラベルからのみ選択（新規作成不可）
   - ファイルパターン: `*.md` → docs, `test/` → test, `.github/` → ci
   - コミットタイプ: `feat` → feature, `fix` → bug, `refactor` → refactor

7. **Draft PR 作成**
   - `gh pr create --draft` で作成

8. **Ready 化（任意）**
   - CI 通過確認後 `gh pr ready` で Review 可能状態に変更

### ブランチ・コミット規則

```
ブランチ: {type}-{subject}     例: feat-user-profile
コミット: {type}: {description}  例: feat: ユーザー認証 API の実装
```

### 制約

- すべての PR は Draft で開始
- 日本語と半角英数字の間に半角スペースを入れる
- テンプレートの構造・HTML コメントを破壊しない
