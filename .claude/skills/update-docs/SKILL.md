# Update Docs

PR（デフォルトブランチとの差分）の変更内容を分析し、影響を受けるドキュメントだけを更新・作成する。狙いはコードとドキュメントの乖離を防ぐことなので、変更に関係しない箇所には手を入れない。

## 手順

### 1. 変更内容の把握

デフォルトブランチとの差分から、変更されたファイル・機能を特定する。

```bash
# デフォルトブランチを検出（取れなければ main を仮定）
base=$(git symbolic-ref refs/remotes/origin/HEAD 2>/dev/null | sed 's@^refs/remotes/origin/@@')
base=${base:-main}
git diff "$base"...HEAD --stat
git diff "$base"...HEAD
```

何が追加・変更・削除され、利用者から見た振る舞いが変わったかを掴む。ドキュメント更新は「読者が知る必要がある変化」を反映する作業なので、内部実装だけの変更ならドキュメント不要なこともある。

### 2. ドキュメント影響の判定

変更ごとに「この変更で古く（嘘に）なる記述はどれか」を起点に、更新が必要なドキュメントを洗い出す。代表的な対象:

- **README**: セットアップ手順、使い方、機能一覧、前提条件
- **CLAUDE.md / AGENTS.md**: コマンド、アーキテクチャ、ワークフロー、規約
- **docs/ 配下**: 設計ドキュメント、ガイド

リポジトリ固有のドキュメント構成があればそれに従う。

### 3. 更新・作成の実行

- 既存ドキュメントは**該当箇所のみ**更新する。無関係な段落や整形には触らない（diff を小さく保ち、レビューしやすくするため）。
- 既存ドキュメントに収まらない新機能・ディレクトリ追加のときだけ、新規ドキュメントを作成する。
- 記述はリポジトリの既存トーン・言語・フォーマットに合わせる。

### 4. 差分の確認

更新したファイルと変更概要をサマリーで提示し、利用者がレビューしてコミット判断できる状態にする。

## 制約

- PR の変更に**関連する部分のみ**更新する。無関係な箇所は触らない（スコープを絞ることで誤更新と無駄な差分を防ぐ）。
- 日本語と半角英数字の間には半角スペースを入れる。
